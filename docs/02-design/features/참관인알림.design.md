## 1. Overview

### 1.1 Design Goals

1. **간결한 구조**: 단일 API Route로 모든 기능 처리 (스크래핑 → 필터링 → 알림)
2. **신뢰성**: 중복 알림 방지, 에러 로깅, 재시도 메커니즘
3. **확장성**: 추후 다른 키워드/게시판 추가 시 쉽게 확장 가능
4. **보안**: 이메일 비밀번호 환경변수로 관리

### 1.2 Design Principles

- **Single Responsibility Principle**: 각 모듈은 하나의 책임만 담당
  - `scraper`: 웹 스크래핑만 담당
  - `emailer`: 이메일 발송만 담당
  - `storage`: KV 저장소 접근만 담당
- **Fail-Safe**: 스크래핑 실패 시 전체 시스템 중단 방지
- **Observable**: 모든 중요 동작은 로그로 남김

## 3. Data Model

### 3.1 Entity Definition

```typescript
// 게시물 엔티티 (스크래핑 결과)
interface BoardPost {
  id: string;           // 게시물 ID (고유)
  title: string;        // 게시물 제목
  url: string;          // 게시물 상세 링크
  author: string;       // 작성자
  date: string;         // 작성일 (YYYY-MM-DD)
  content?: string;     // 내용 (옵션 - 기본 내용은 미포함)
}

// 이메일 발송 결과
interface EmailResult {
  success: boolean;
  postId: string;
  error?: string;
}

// KV 저장소 키 형식
interface StorageKeys {
  processed: `processed:${postId}`;      // 처리한 게시물 ID
  lastRun: 'last_run';                   // 마지막 실행 시간
}
```

### 3.2 Vercel KV Schema

Vercel KV는 Key-Value 저장소이므로 다음 형식으로 저장:

| Key | Value | TTL | Purpose |
|-----|-------|-----|---------|
| `processed:{postId}` | `1` | 30 days | 해당 게시물 알림 처리 표시 |
| `last_run` | ISO timestamp | - | 마지막 실행 시간 로깅 |

### 3.3 Data Transformations

```
HTML (선관위 게시판)
    ↓ Cheerio 파싱
BoardPost[] (게시물 목록)
    ↓ "참관인" 필터링
BoardPost[] (필터링된 게시물)
    ↓ KV 중복 체크
BoardPost[] (신규 게시물만)
    ↓ 이메일 발송
EmailResult[]
```

## 4. API Specification

### 4.1 Endpoint List

| Method | Path | Description | Auth | Cron |
|--------|------|-------------|------|:----:|
| GET | /api/check-board | 게시판 확인 및 알림 발송 | Not Required | ✅ Daily |
| GET | /api/health | 건강 상태 확인 | Not Required | - |

### 4.2 Detailed Specification

#### `GET /api/check-board`

**요약**: 매일 실행되어 게시판을 확인하고 "참관인" 포함 게시물을 이메일로 알림

**Request Headers:**
```http
Authorization: Bearer {CRON_SECRET}  # Vercel Cron verification
```

**Response (200 OK):**
```json
{
  "success": true,
  "timestamp": "2026-04-25T00:00:00Z",
  "summary": {
    "totalPosts": 5,
    "filteredPosts": 2,
    "newPosts": 1,
    "emailsSent": 1,
    "errors": 0
  },
  "newPosts": [
    {
      "id": "123456",
      "title": "제xx대 선거 참관인 모집 공고",
      "url": "https://su.nec.go.kr/..."
    }
  ]
}
```

**Error Responses:**
- `500 Internal Server Error`: 스크래핑 실패, KV 연결 실패 등
- `401 Unauthorized`: Cron secret 불일치 (선택적 보안)

#### `GET /api/health`

**요약**: 시스템 상태 확인

**Response (200 OK):**
```json
{
  "status": "healthy",
  "timestamp": "2026-04-25T12:00:00Z",
  "lastCheck": "2026-04-25T00:00:00Z",
  "kvConnected": true
}
```

## 5. Error Handling

### 5.1 Error Code Definition

| Code | Message | Cause | Handling |
|------|---------|-------|----------|
| SCRAPE_FAILED | 스크래핑 실패 | 네트워크 오류, HTML 변경 | 로그 남기고 종료, 알림 발송 안 함 |
| PARSE_FAILED | 파싱 실패 | HTML 구조 변경 | 로그 남기고 에러 메일 발송 |
| KV_FAILED | KV 연결 실패 | Vercel KV 장애 | 로그 남기고 계속 진행 (중복 체크 스킵) |
| EMAIL_FAILED | 이메일 발송 실패 | SMTP 오류 | 로그 남기고 재시도 (최대 3회) |

### 5.2 Error Response Format

```typescript
interface ErrorResponse {
  success: false;
  error: {
    code: string;
    message: string;
    details?: unknown;
  };
  timestamp: string;
}
```

## 9. Coding Convention Reference

### 9.1 Naming Conventions

| Target | Rule | Example |
|--------|------|---------|
| Functions | camelCase, 동사 시작 | `scrapeBoard()`, `sendEmail()`, `checkDuplicate()` |
| Constants | UPPER_SNAKE_CASE | `BOARD_URL`, `TARGET_KEYWORD` |
| Types/Interfaces | PascalCase | `BoardPost`, `EmailResult` |
| Files | camelCase.ts | `scraper.ts`, `emailer.ts` |

### 9.2 Import Order

```typescript
// 1. External libraries
import cheerio from 'cheerio';
import nodemailer from 'nodemailer';

// 2. Internal absolute imports
import { scrapeBoard } from '@/lib/scraper';
import { sendEmail } from '@/lib/emailer';

// 3. Type imports
import type { BoardPost } from '@/types';

// 4. Environment variables
const BOARD_URL = process.env.BOARD_URL!;
```

### 9.3 Environment Variables

| Variable | Purpose | Scope | Example |
|----------|---------|-------|---------|
| `BOARD_URL` | 게시판 URL | Server | `https://su.nec.go.kr/...` |
| `TARGET_KEYWORD` | 검색 키워드 | Server | `참관인` |
| `GMAIL_EMAIL` | 발신 Gmail 주소 | Server | `sender@gmail.com` |
| `GMAIL_APP_PASSWORD` | Gmail 앱 비밀번호 | Server | `xxxx xxxx xxxx xxxx` |
| `ALERT_TO_EMAIL` | 수신 이메일 | Server | `receiver@gmail.com` |
| `KV_REST_API_URL` | Vercel KV URL | Server | `https://...` |
| `KV_REST_API_TOKEN` | Vercel KV 토큰 | Server | `xxxxx...` |
